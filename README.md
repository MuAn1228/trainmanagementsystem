## 功能总览

| 模块 | 功能 | 数据结构 |
|------|------|----------|
| 车次查询 | 按出发站/到达站模糊搜索 | 数组遍历 |
| 精确查询 | 按车次号 O(1) 查找 | 哈希表（链地址法） |
| 多维度排序 | 按时/耗时/票价/到达时间排序 | 快速排序 / 归并排序 |
| 最优路径 | 两站之间最短时间/最少换乘 | 图（邻接表）+ Dijkstra |
| 车票预订 | 选车次 → 选席别 → 实名下单 | 单链表（头插法） |
| 退票 | 按订单号退票，退 80% 票款 | 链表查找 + 座位回收 |
| 改签 | 更换车次/席别，差额多退少补 | 链表查找 + 重新下单 |
| 订单查询 | 查看所有订单及状态 | 单链表遍历 |
| 候补队列 | 无票时登记候补，退票自动兑付 | 链式队列（FIFO） |
| 数据持久化 | 列车/订单信息文件读写 | 文件 I/O |

## 数据结构一览
```

数据结构           应用场景                      
数组              Train[] 车次存储、遍历查询     
哈希表（链地址）    车次号 → 数组下标 O(1) 查找

文件 I/O          trains.txt / orders.txt 持久化 
```

## 数据格式

### 列车数据 (`trains.txt`)

每条记录包含：车次号、出发站、到达站、发车时间、到达时间、耗时(分钟)、总座位数，以及硬座/软座/硬卧/软卧四种席别的票价和余票。

```
G23  广州  深圳  2026:06:20:15:30  2026:06:20:16:00  30  850  40.0  250  80.0  150  120.0  300  200.0  185
```

### 订单数据 (`orders.txt`)

每条记录包含：订单号、车次、姓名、身份证号、席别、座位号、票价、状态、下单时间。

## 核心算法

### Dijkstra 最短路径

将所有站点构建为带权有向图（邻接表），边权为列车耗时。支持：
- **最短时间**：求总耗时最小的路径
- **最少换乘**：求换乘次数最少的路径

```
最优路径规划展示：
总耗时: 420 分钟
路径: 拉萨 --(Z21)--> 北京 --(G25)--> 大连
共 3 站，1 次换乘
```

### 哈希查找

车次号 → 哈希函数 → 桶内链表遍历 → 得到数组下标，平均时间复杂度 **O(1)**。

### 候补队列

无票时乘客进入候补队列（FIFO）；有人退票时，系统自动通知队首乘客，实现"退票即兑付"。
